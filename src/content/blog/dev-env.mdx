As stereotypical as it is, my development machine really is a second home. I like it to be comfortable and efficient ~~and have a bidet in every room~~.

Here's my machine configuration, as well as my desk setup. For my mobile-specific environment, [go here](/blog/mobile-env). <a target="_blank" rel="noopener noreferrer" href="https://github.com/gretzky/dotfiles">Dotfiles repository</a>.

## Styles

Across the board, I use the <a target="_blank" rel="noopener noreferrer" href="https://github.com/ayu-theme/ayu-colors">Ayu</a> color scheme (usually 'mirage') and the [Hack Nerd Font](https://www.nerdfonts.com/).

## Terminal

I live in my terminal. I use **<a target="_blank" rel="noopener noreferrer" href="https://github.com/jwilm/alacritty">Alacritty</a>**.

**Shell**: ZSH with <a target="_blank" rel="noopener noreferrer" href="https://starship.rs/">starship prompt</a>

**Plugin management**: <a target="_blank" rel="noopener noreferrer" href="https://github.com/zplug/zplug">zplug</a>

**Filesystem navigation**: <a target="_blank" rel="noopener noreferrer" href="https://github.com/rupa/z">z</a>, <a target="_blank" rel="noopener noreferrer" href="https://github.com/junegunn/fzf">fzf</a>, and <a target="_blank" rel="noopener noreferrer" href="https://github.com/athityakumar/colorls">color-ls</a>.

**Multiplexer**: <a target="_blank" rel="noopener noreferrer" href="https://github.com/tmux/tmux">tmux</a>

For in-terminal editing, I use <a target="_blank" rel="noopener noreferrer" href="https://neovim.io/">neovim</a> with <a target="_blank" rel="noopener noreferrer" href="https://github.com/junegunn/vim-plug">vim-plug</a> to handle the following plugins:

- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/junegunn/fzf.vim">fzf</a>**: fuzzy file finder
- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/junegunn/goyo.vim">goyo</a>**: distraction-free writing
- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/junegunn/limelight.vim">limelight</a>**: focus on the current paragraph
- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/itchyny/lightline.vim">lightline</a>**: status line
- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/dense-analysis/ale">ale</a>**: linting
- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/preservim/nerdtree">nerdtree</a>**: file explorer
- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/preservim/nerdcommenter">nerdcommenter</a>**: comment toggling
- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/mhinz/vim-signify">signify</a>**: version control indicators
- **<a target="_blank" rel="noopener noreferrer" href="https://github.com/voldikss/vim-floaterm">floaterm</a>**: terminal in a floating window

## Text Editor

When not in the terminal, I'm using <a target="_blank" rel="noopener noreferrer" href="https://code.visualstudio.com/">VSCode</a>.

Aside from basic language extensions, I use the following extensions heavily:

- Auto Close Tag
- Auto Complete Tag
- Auto Rename Tag
- Code Spell Checker
- GitHub Copilot
- GitLens
- IntelliCode
- Path Intellisense
- TODO Highlight

## Tooling

I use <a target="_blank" rel="noopener noreferrer" href="https://brew.sh/">Homebrew</a> to manage my system packages.

### UNIX

I've added all of the UNIX and GNU ports, since I prefer Linux utilities over the BSD ones that come with macOS.

- **ack**: Enhanced grep for code searching
- **binutils**: GNU version of binary tools
- **ccat**: cat with syntax highlighting
- **coreutils**: essential GNU file and shell tools
- **diffutils**: tools for comparing files
- **findutils**: tools for searching files
- **gawk**: GNU version of the awk text processor
- **gnu-sed**: GNU version of the sed stream editor
- **gnu-tar**: GNU version of the tar archive tool
- **gnu-time**: GNU version of the time command
- **gnu-which**: GNU version of the which command
- **gnupg**: GNU tools for encryption and signing
- **grep**: text search utility
- **gzip**: file compression utility
- **moreutils**: additional Unix tools
- **rename**: utility for batch renaming files
- **rsync**: file synchronization utility
- **tree**: directory structure visualizer
- **wget**: file downloader

### Language Version Managment

For the more finicky languages (especially on macOS), I use the following version managers:

**Node**: <a target="_blank" rel="noopener noreferrer" href="https://github.com/tj/n">n</a>

**Ruby**: <a target="_blank" rel="noopener noreferrer" href="https://github.com/postmodern/chruby) with [ruby-install](https://github.com/postmodern/ruby-install">chruby</a>

**Python**: <a target="_blank" rel="noopener noreferrer" href="https://github.com/pyenv/pyenv">pyenv</a>
